# Инструкция: Автономная сборка для работы без интернета

## Проблема
Стандартная сборка (`Скачать → Скачать билд`) включает скрипты poehali.dev и требует интернет-соединения.

## Решение
Создана специальная автономная сборка без внешних зависимостей.

## Как собрать автономную версию

### Шаг 1: Подключить GitHub
1. В редакторе poehali.dev: **Скачать → Подключить GitHub**
2. Авторизуйтесь и выберите репозиторий
3. Код автоматически загрузится в GitHub

### Шаг 2: Склонировать проект на компьютер
```bash
git clone https://github.com/ВАШ_АККАУНТ/itc-siberia-website.git
cd itc-siberia-website
```

### Шаг 3: Установить зависимости
```bash
npm install
# или
bun install
```

### Шаг 4: Собрать автономную версию
```bash
npx vite build --config vite.config.standalone.ts --base ./ --mode production
```

Эта команда:
- ✅ Использует специальную конфигурацию без внешних скриптов
- ✅ Создаёт папку `dist-standalone/` с готовым сайтом
- ✅ Использует относительные пути (работает с `file://`)
- ✅ Убирает скрипты аналитики и телеметрии

### Шаг 5: Протестировать локально
```bash
# Открыть папку dist-standalone
cd dist-standalone

# Просто откройте index.html двойным кликом!
```

## Что будет работать

✅ **Работает без интернета:**
- Навигация по страницам
- Все компоненты и стили
- Формы (через mailto)
- Адаптивный дизайн

⚠️ **Требует интернет:**
- Яндекс.Карты (встроенные на странице Контакты)
- Форма обратной связи через PHP (на хостинге)

## Загрузка на хостинг

Для работы форм на хостинге:
1. Возьмите файлы из `dist-standalone/`
2. Добавьте `send-contact-form.php` из обычного билда
3. Измените email в PHP-файле
4. Загрузите на REG.RU в `public_html/`

## Отличия от обычного билда

| Параметр | Обычный билд | Автономный билд |
|----------|-------------|-----------------|
| Скрипты poehali.dev | ✅ Есть | ❌ Удалены |
| Яндекс.Метрика | ✅ Есть | ❌ Удалена |
| Работает offline | ⚠️ Частично | ✅ Полностью |
| Base path | `/` | `./` |
| Конфиг | vite.config.ts | vite.config.standalone.ts |

## Альтернатива: Быстрая модификация

Если нет возможности собрать локально, можно вручную удалить скрипты из index.html в обычном билде:

1. Скачать обычный билд
2. Открыть `dist/index.html`
3. Удалить строки со скриптами:
   ```html
   <!-- Удалить эти строки -->
   <script src="https://cdn.poehali.dev/intertnal/js/pp-min-2.js"></script>
   <script src="https://cdn.poehali.dev/intertnal/js/route-min.js"></script>
   <script src="https://cdn.poehali.dev/intertnal/js/telemetry-min.js"></script>
   <script src="https://cdn.poehali.dev/intertnal/js/inspector-min.js"></script>
   
   <!-- И Яндекс.Метрику -->
   <script type="text/javascript">...</script>
   ```
4. Сохранить и открыть

⚠️ **Внимание:** Пути к ресурсам должны быть относительными (`./assets/...`)
